Falsy and Truthy Values


Javascript tiene valores booleanos: true y false
que funcionan normal, por ejemplo: 

								
true + 1;
true == 1;
Con algunos operadores true y false actúan como 1 y 0

Falsy and Truthy Values


Todos los valores tienen una "representación" booleana 

								
if("Esto es un string") {
	console.log('En JavaScript, esto es legal);
}
Eso NO quiere decir que "Un Texto" == true

Los siguientes valores son "Falsy"
false - 0 - '' - NaN - null - undefined

El resto son "Truthy", incluyendo
"false" y "0"

Falsy and Truthy Values


								
//Ejemplo 1
if (5) {
	console.log('5 en un valor truthy');
}

//Ejemplo 2
if (0) {
	console.log('0 NO es un valor truthy');
} else {
	console.log('0 es un valor falsy');
}

//Ejemplo 3
if (['array', 'de', 'strings']) {
	console.log('Los arrays también son truthy');
}

Operadores

=== vs. ==

Type Coercion

Cuando se usa == uno de los dos valores es convertido antes de la comparación
Hay reglas para la conversión. La mayoría convierte a número uno de los valores 

								
1 == true;
'0.1' == 0.1;
'hola mundo' == ['hola mundo'];
null == undefined;
null == null;
undefined == undefined;

=== vs. ==

Type Coercion

Cuando se usa === los valores deben ser del mismo tipo, de lo contrario retorna false 

								
1 === true;
'0.1' === 0.1;
'hola mundo' === ['hola mundo'];
null === undefined;
null === null;
undefined === undefined;

&&

'El operador Guardián'

En un if funciona igual que en Java o .Net, como un AND 

								
if (true && false) {
 console.log('Si esto se imprime no vuelvo, en mi vida, a programar!');
}

if (true && true) {
 console.log('&& funciona como un operador AND normal');
}

&&

'El operador Guardián'

En la inicialización de variables, && es un guardián

Si el primer valor es Truthy, retorna el segundo
Si el primer valor es Falsy, retorna el primero

								
//Lanza una excepción
var guardOperator = window.Person.name; 

//No lanza la excepción, simplemente le asigna window.Person
//o sea undefined
var guardOperator = window.Person && window.Person.name; 

//Ahora window.Person.name sí tiene un valor
window.Person = {name: "Pam"};

//Ya no asigna window.Person, sino window.Person.name
var guardOperator = window.Person && window.Person.name;

||

'El operador por Defecto'

En un if funciona igual que en Java o .Net, como un OR 

								
if (false || false) {
 console.log('Si esto se imprime no vuelvo, en mi vida, a programar!');
}

if (true || false) {
 console.log('|| funciona como un operador OR normal');
}

||

'El Operador por Defecto'

Inicializando una variable, || es un operador por defecto

Si el primer valor es Truthy, retorna el Primero
Si el primer valor es Falsy, retorna el Segundo

							
//Si el primer valor es undefined, entonces use el valor "Por defecto"
var defaultValue = window.noValue || "This is my default Value";

//Si el primer valor existe, no toma el valor por defecto
window.aValue = "House";
var defaultValue = window.aValue || "This is my default Value";

for

Hay dos tipos de for:
El que todos conocemos 

								
var vector = ['esto', 'es', 'un', 'vector'];
for (var i = 0; i < vector.length; i++)  {
	console.log(vector[i]);
}
Y otro que itera sobre las propiedades de un objeto, incluyendo las heredadas 
								
var casa = {color:'rojo', portero:'false', telefono:'23123213'};
for (var atributo in casa)  {
	console.log(atributo);
}
