MV*

MV Whatever.

Con la evolución de la Web, han evolucionado también las aplicaciones Web.

Después de la aparación de HTML5, el cliente, que antes era simple, se ha vuelto robusto y complejo.

Es por esto que han aparecido frameworks que pretenden organizar lo caótico que puede ser el código del cliente. Entre estos se encuentran Backbone, Angular, Knockout

Estos frameworks son, algo así como, un sabor del famoso MVC, sin serlo completamente.
Por eso se conoce el tema como MV* o MVWhatever.

MVC

¿Cuál era el Problema?

Los patrones existen para resolver problemas específicos.

El patrón MVC (Model-View-Controller) busca mejorar:

  • La mantenibilidad del código en la capa de presentación
  • Abstraer las responsabilidades: lógica de negocio mezclada con el UI
  • Reuso de código

MVC

Model View Controller

MVC es un patrón de arquitectura.

Algunos de los patrones de diseño usados en MVC son:

  • Observer: Cuando cambia el modelo, la vista debe ser modificada y viceversa.
  • Composite: Una vista puede estar compuesta de otras vistas (una página que contiene widgets).
  • Delegate: La vista delega las acciones al controlador.
